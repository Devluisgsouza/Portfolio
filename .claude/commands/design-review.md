## O que analisar

### Hierarquia Visual
- Tipografia: tamanhos, pesos e escala entre headings/body
- Espaçamento: consistência de margins/paddings e ritmo vertical
- Contraste de cores: ratios de acessibilidade (WCAG AA mínimo)
- Alinhamento e grid: consistência da grade

### Identidade Visual
- Consistência de paleta de cores (CSS custom properties)
- Uso das fontes (Syne, DM Mono, DM Sans)
- Iconografia coerente
- Tom visual geral (moderno, minimalista, dev-focused)

### Componentes UI
- Buttons: estados hover/focus/active
- Cards: sombras, bordas, espaçamento interno
- Navegação: clareza e feedback visual
- Animações: suavidade, propósito, não excessivas

### Landing Page
- Above the fold: impacto imediato, CTA claro
- Fluxo de leitura: F-pattern ou Z-pattern
- Seções: transição entre seções, espaçamento
- Footer: completude das informações

## Como reportar
Para cada problema encontrado:
1. **Local**: arquivo + seletor CSS ou elemento HTML
2. **Problema**: o que está errado e por quê
3. **Solução**: código concreto para corrigir

Priorize por impacto visual. Liste no máximo 8 problemas mais relevantes.
